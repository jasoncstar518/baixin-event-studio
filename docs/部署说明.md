## 系统要求

### 硬件要求
- **CPU:** 2 核心以上
- **内存:** 4GB 以上（推荐 8GB）
- **磁盘:** 10GB 可用空间

### 软件要求
- **Docker:** 20.10+
- **Docker Compose:** 2.0+
- **Node.js:** 18.x+（手动部署需要）
- **PostgreSQL:** 15+（手动部署需要）
- **Redis:** 7+（手动部署需要）

## Docker 部署（推荐）

### 1. 克隆项目

```bash
git clone https://github.com/jasoncstar518/baixin-event-studio.git
cd baixin-event-studio
```

### 2. 配置环境变量

```bash
# 后端环境配置
cp backend/.env.example backend/.env
# 编辑 backend/.env，修改关键配置
```

**关键配置项：**
```bash
# 生产环境务必修改
JWT_SECRET="your-super-secret-key-$(openssl rand -hex 32)"
NODE_ENV=production

# 数据库连接（Docker 环境使用容器名）
DATABASE_URL="postgresql://baixin:baixin123@postgres:5432/baixin_event_studio"
REDIS_URL="redis://redis:6379"
```

### 3. 启动服务

```bash
# 一键启动所有服务
docker-compose up -d

# 查看运行状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 4. 数据库初始化

```bash
# 执行数据库迁移
docker-compose exec backend npm run prisma:migrate

# 生成 Prisma 客户端
docker-compose exec backend npm run prisma:generate
```

### 5. 访问服务

| 服务 | 地址 | 说明 |
|------|------|------|
| 前端 | http://localhost:5173 | 可视化编辑器 |
| 后端 API | http://localhost:3000 | RESTful API |
| PostgreSQL | localhost:5432 | 数据库（外部访问） |
| Redis | localhost:6379 | 缓存（外部访问） |

### 6. 停止服务

```bash
# 停止所有服务
docker-compose down

# 停止并删除数据卷（谨慎使用！）
docker-compose down -v
```

## 手动部署

### 1. 安装依赖

#### 安装 Node.js
```bash
# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# macOS
brew install node@18
```

#### 安装 PostgreSQL
```bash
# Ubuntu/Debian
sudo apt-get install postgresql-15 postgresql-contrib

# macOS
brew install postgresql@15
```

#### 安装 Redis
```bash
# Ubuntu/Debian
sudo apt-get install redis-server

# macOS
brew install redis
```

### 2. 克隆项目

```bash
git clone https://github.com/jasoncstar518/baixin-event-studio.git
cd baixin-event-studio
```

### 3. 配置数据库

```bash
# 创建数据库
sudo -u postgres psql
CREATE DATABASE baixin_event_studio;
CREATE USER baixin WITH PASSWORD 'baixin123';
GRANT ALL PRIVILEGES ON DATABASE baixin_event_studio TO baixin;
\q
```

### 4. 部署后端

```bash
cd backend

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件

# 生成 Prisma 客户端
npm run prisma:generate

# 执行数据库迁移
npm run prisma:migrate

# 启动服务（开发环境）
npm run dev

# 启动服务（生产环境）
npm start
```

### 5. 部署前端

```bash
cd frontend

# 安装依赖
npm install

# 配置 API 地址
# 编辑 .env 或 .env.production
VITE_API_URL=http://your-server.com:3000

# 开发环境启动
npm run dev

# 生产环境构建
npm run build

# 预览构建结果
npm run preview
```

### 6. 使用 PM2 管理进程（生产环境）

```bash
# 安装 PM2
npm install -g pm2

# 启动后端
cd backend
pm2 start src/index.js --name baixin-backend

# 启动前端（构建后）
cd frontend
pm2 serve dist/ 5173 --name baixin-frontend

# 查看状态
pm2 status

# 保存配置（开机自启）
pm2 save
pm2 startup
```

## 生产环境配置

### 1. 域名配置

```nginx
# Nginx 配置示例
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /var/www/baixin-frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 2. HTTPS 配置（Let's Encrypt）

```bash
# 安装 Certbot
sudo apt-get install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

### 3. 环境变量安全

```bash
# 生产环境 .env 示例
NODE_ENV=production
PORT=3000

# 使用强随机密钥
JWT_SECRET="$(openssl rand -hex 32)"
JWT_EXPIRES_IN="7d"

# 数据库连接
DATABASE_URL="postgresql://user:strong_password@localhost:5432/baixin_event_studio"

# Redis 连接
REDIS_URL="redis://localhost:6379"

# 文件存储（阿里云 OSS）
OSS_ACCESS_KEY_ID="your-access-key"
OSS_ACCESS_KEY_SECRET="your-access-secret"
OSS_BUCKET="baixin-event"
OSS_REGION="oss-cn-hangzhou"
```

### 4. 数据库备份

```bash
#!/bin/bash
# backup.sh - 数据库备份脚本

BACKUP_DIR="/var/backups/baixin"
DATE=$(date +%Y%m%d_%H%M%S)
PGPASSWORD="baixin123" pg_dump -h localhost -U baixin baixin_event_studio > $BACKUP_DIR/backup_$DATE.sql

# 保留最近 7 天备份
find $BACKUP_DIR -name "backup_*.sql" -mtime +7 -delete
```

```bash
# 添加定时任务
crontab -e
# 每天凌晨 2 点备份
0 2 * * * /path/to/backup.sh
```

### 5. 监控与日志

```bash
# 查看实时日志
docker-compose logs -f

# PM2 日志
pm2 logs baixin-backend

# 系统资源监控
pm2 monit

# 安装 PM2 Plus（云端监控）
pm2 plus
```

## 常见问题

### Q1: 容器启动失败
```bash
# 检查 Docker 状态
docker-compose ps

# 查看详细日志
docker-compose logs backend

# 重建容器
docker-compose up -d --force-recreate
```

### Q2: 数据库连接失败
```bash
# 检查数据库是否运行
docker-compose ps postgres

# 测试连接
docker-compose exec postgres pg_isready -U baixin

# 查看数据库日志
docker-compose logs postgres
```

### Q3: 前端无法访问后端 API
- 检查 `VITE_API_URL` 配置是否正确
- 确认 CORS 设置（后端允许前端域名访问）
- 检查防火墙规则

### Q4: Prisma 迁移错误
```bash
# 重置数据库（开发环境）
docker-compose exec backend npx prisma migrate reset

# 手动修复迁移
docker-compose exec backend npx prisma migrate resolve --applied "migration_name"
```

### Q5: 内存不足
```bash
# 限制容器内存
# 修改 docker-compose.yml
services:
  backend:
    deploy:
      resources:
        limits:
          memory: 512M
```

## 快速部署脚本

```bash
#!/bin/bash
# deploy.sh - 一键部署脚本

# 1. 克隆项目

docker-compose exec backend npm run prisma:migrate

echo "✅ 部署完成！"

echo "API 地址：http://$(hostname -I | awk '{print $1}'):3000"
```
